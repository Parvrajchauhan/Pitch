## ΏθΥΝ What Is This?

**Pitch Visualizer** is a full-stack AI service that transforms a block of narrative text έΑΦ a customer success story, a startup pitch, a product description έΑΦ into a **coherent, multi-panel visual storyboard**.

It does three things well:
1. **Segments** your text into logical scenes using NLP sentence tokenization (NLTK)
2. **Engineers** a rich, visually descriptive prompt for each scene using an LLM (Gemini)
3. **Generates** a unique image per panel via a text-to-image API and presents everything as a clean, sequential storyboard UI

 It is a complete **idea έΗΤ narrative έΗΤ visuals** pipeline.

## έερ Features
- Ώθπι **NLP-based sentence segmentation** έΑΦ splits narrative into meaningful, standalone scenes
- ΏθΟρ **LLM-powered prompt refinement** έΑΦ Gemini transforms each sentence into a richly descriptive visual prompt, adding cinematic framing, lighting, and mood
- ΏθΨ╝Ύ╕Π **AI image generation** έΑΦ calls a text-to-image API (Stable Diffusion / DALL┬╖E) for each engineered prompt
- ΏθΟηΎ╕Π **Multi-panel storyboard UI** έΑΦ panels are rendered sequentially with original text captions
- ΏθΟφ **User-selectable visual styles** έΑΦ choose from `cinematic`, `anime`, `Fantasy`, `realistic`, or `Sketch` before generating
- ΏθΡ│ **Fully containerized** έΑΦ Docker Compose brings up backend + frontend with a single command

## έγβΎ╕Π Setup & Installation

### Prerequisites
- Docker & Docker Compose, **or** Python 3.10+ and Node.js 18+
- A Gemini API key ([get one free](https://aistudio.google.com/app/apikey))
- A HuggingFace API key for image generation

### 1. Clone the Repository

```bash
git clone https://github.com/Parvrajchauhan/Pitch.git
cd Pitch
```

### 2. Configure Environment Variables

```bash
cp server/.env.example server/.env
```

Edit `server/.env`:

```env
GEMINI_API_KEY=
GEMINI_MODEL=gemini-3-flash-preview
HF_API_KEY=
```

### 3A. Run with Docker (Recommended)

```bash
docker-compose up --build
```

- **Backend:** http://localhost:8000
- **Frontend:** http://localhost:3000
- **API Docs:** http://localhost:8000/docs

### 3B. Run Manually

**Backend:**
```bash
cd server
pip install -r requirements.txt
python -m nltk.downloader punkt punkt_tab
uvicorn api.main:app --reload --port 8000
```

**Frontend:**
```bash
cd client
npm install
npm run dev
```

## Ώθπι Design & Architecture

### Pipeline

```
Raw Text Input
     έΦΓ
     έΨ╝
[1] Gemini LLM έΑΦ Visual Thread Generator
     έΦΓ  Extracts consistent elements (characters, setting, tone, color palette)
     έΨ╝
[2] NLTK Sentence Tokenizer
     έΦΓ  Splits into έΚξ3 scene segments
     έΨ╝
[3] Gemini LLM έΑΦ Prompt Engineer
     έΦΓ  Adds visual framing, lighting, mood, style keywords (using visual thread)
     έΨ╝
[4] Text-to-Image API (Stability AI / DALL┬╖E)
     έΦΓ  Generates one image per engineered prompt
     έΨ╝
[5] Storyboard Renderer (React)
     έΦΓ  Sequential panels, captions, style badge
     έΨ╝
Visual Storyboard Output
```

### Prompt Engineering Strategy

Each sentence is sent to Gemini with a system instruction to return a single, highly descriptive visual prompt. The instruction enforces:
- **Scene framing:** wide shot, close-up, aerial, etc.
- **Lighting descriptors:** golden hour, dramatic shadows, neon, etc.
- **Mood/atmosphere:** tense, hopeful, ethereal, etc.
- **Style keywords** appended from the user's selected preset (e.g., `"cinematic, 35mm film, high detail"`)
